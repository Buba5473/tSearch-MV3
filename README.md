# tSearch (Manifest V3 Edition)

**tSearch** — это быстрое, безопасное и энергоэффективное браузерное расширение на движке Chromium для одновременного параллельного мультипоиска выделенного текста по множеству поисковых систем и торрент-трекеров.

Этот проект представляет собой полную и глубокую модернизацию оригинального плагина Feverqwe. Весь код адаптирован под жесткие стандарты **Manifest V3 (MV3)**, избавлен от архитектурных ограничений старых версий и дополнен мощным реактивным интерфейсом фильтрации результатов.

---

## 🛠 Аккумулированный список изменений и оптимизаций

В ходе адаптации и оптимизации плагина были реализованы следующие ключевые изменения:

### 1. Архитектура и Безопасность (Переход на Manifest V3)
* **Миграция на Service Worker (`background.js`)**: Фоновый процесс полностью переведен на событийно-ориентированную модель. Он больше не удерживает оперативную память браузера в простое, а «просыпается» реактивно — только в момент вызова контекстного меню или запроса скрапинга.
* **Полный отказ от `eval()` (Strict CSP)**: Из проекта полностью удален устаревший движок `exKit`, выполнявший строки кода динамически. Извлечение данных теперь работает на безопасных декларативных CSS-селекторах через изолированное Chromium API `chrome.scripting.executeScript`.
* **Модульная структура (ES6 Modules)**: Благодаря директиве `"type": "module"` в манифесте, конфигурации трекеров и парсеров вынесены в отдельные изолированные файлы-модули с использованием операторов `import/export`.

### 2. Масштабное расширение базы трекеров (Интеграция 38+ источников)
* **Объединение баз данных**: В единый декларативный реестр `trackerRegistry.js` импортированы и структурированы все трекеры из оригинальной папки `external`, текстовых списков, а также логика регулярных выражений из базы **TorrentMonitor** (`korshun/TorrentMonitor`) и **Jackett**.
* **Исправление ошибок и дубликатов**: Скорректированы зеркала, удалены неактуальные дубли, исправлены специфические опечатки в названиях и раскладках (например, *ЕруЗшкфеИфн* корректно переведена в *The Pirate Bay*).
* **Поддержка нишевого контента**: Встроены точные селекторы для сериальных и аниме-порталов (**LostFilm.tv**, **AniLibria**, **Nyaa.si**), аудиокниг (**Abook-Club**) и тяжелой музыки (**Metal-Tracker**).

### 3. Интеллектуальный обход защит и кэширование сессий
* **Безопасная симуляция POST-запросов**: Доступ к закрытым поисковым формам форумов (RuTracker, Rustorka, Tapochek) реализован через автоматическую фоновую генерацию и отправку скрытых HTML-форм.
* **Обнаружение капчи и авторизации**: Если трекер заблокирован проверкой Cloudflare/SmartCaptcha или у пользователя «слетела» сессия, `background.js` плавно выводит эту скрытую вкладку из тени на передний план и присылает системное уведомление.
* **Автоматический кэш кук**: После ручного ввода пароля или одного клика по капче, Chromium намертво кэширует сессионный токен в Cookies. Вкладка-донор автоматически скрывается и закрывается, а результаты поиска бесшовно улетают в общую таблицу.

### 4. Новый двухпанельный интерфейс результатов (`search.html` / `search.js`)
* **Спецификация роутинга**: Поиск открывает внутреннюю страницу расширения по аналогии с современными SPA-приложениями: `chrome-extension://[ID]/search.html?query=Miller%27s+Crossing`.
* **Левая панель (Интерактивный сайдбар)**:
  * *Ключевые и стоп-слова*: Мгновенное включение нужных релизов (например, «1080p») и исключение мусора (например, «3D») на лету без повторного опроса сайтов.
  * *Точный размер (ГБ)*: Пользовательские поля ввода диапазона размера «От» и «До» с шагом `0.1`.
  * *Умный фильтр дат*: Возможность отсечь старые раздачи, выбрав срез («За последние 24 часа», «Неделю», «Месяц»).
  * *Интерактивные табы*: Трекеры выводятся кнопками с счетчиками найденных торрентов. Клик по табу изолирует результаты этого сайта, повторный клик — возвращает полный список.
  * *Кнопки входа*: Возле каждого трекера выводится маркер `🟢` (авторизован) или `🔑` (требуется вход). Клик по ключу открывает страницу авторизации сайта.
* **Правая панель (Сводная таблица результатов)**:
  * Из таблицы убраны лишние элементы, заголовок раздачи стал чистым текстом, а под ним выводится аккуратная **гиперссылка-хлебные крошки** с путем категории (например, *Кино » Зарубежное кино*), ведущая на саму тему форума.
  * Добавлена **динамическая сортировка колонок** (Название, Размер, Сиды, Личи) по клику на заголовки с отображением стрелочек `▲`/`▼`. Сортировка по размеру точна, так как текстовые строки (ГБ/МБ) утилитарно переводятся в чистые байты (`sizeBytes`).
  * **Цветная индикация возраста**: Перед названием релиза выводится стилизованный бейдж. Свежие раздачи подсвечиваются мягким зеленым (`[Сегодня]`) и синим (`[Вчера]`) цветами, архивные — нейтральным серым (`[5 мес.]`, `[2 г.]`).

### 5. Система динамического обновления парсеров (No-Code Update)
* В плагин внедрен гибридный механизм синхронизации. Локальные селекторы в `trackerParsers.js` выступают резервной базой.
* По таймеру (раз в 6 часов через Alarms API) или принудительно по кнопке из Popup, расширение асинхронно скачивает актуальный декларативный JSON-файл с удаленного сервера (`REMOTE_PARSER_URL`) и кэширует его в `chrome.storage.local`.
* Если трекер изменит верстку или классы таблиц, разработчику достаточно отредактировать удаленный JSON-файл на сервере (например, на GitHub Raw) — у всех пользователей парсеры починятся **автоматически без переустановки расширения**.

### 6. Кросс-платформенный скрипт автоматической сборки (`build.sh`)
* Написан процедурный Bash-скрипт для окружения **MSYS2 (UCRT64)**.
* **Менеджер зависимостей**: Скрипт сам проверяет наличие `zip`, `openssl` и `curl` и при их отсутствии интерактивно запрашивает разрешение на установку через `pacman`.
* **Автозагрузка графики**: Скрипт самостоятельно создает директорию `icons/` и скачивает туда оригинальные PNG-иконки бинокля во всех трех разрешениях по прямым ссылкам.
* **Кросс-компиляция**: Скрипт принимает аргумент целевой операционной системы (`windows` или `linux`, по умолчанию `windows`). Он автоматически транслирует пути файловой системы и собирает в папку `build_dist/` готовый `.zip` архив для Web Store и подписанный бинарный пакет `.crx` с автоматической генерацией приватного ключа подписи `.pem`.

## 📂 Структура проекта расширения

```text
tSearch-MV3/
├── manifest.json          # Конфигурация, системные разрешения (scripting, alarms, notifications)
├── background.js          # Фоновый Service Worker (диспетчер форм, скрытых вкладок и автообновлений)
├── trackerRegistry.js     # База данных: URL-адреса, методы передачи (GET/POST) и удаленный сервер правил
├── trackerParsers.js      # Базовые декларативные CSS-селекторы таблиц и утилиты нормализации данных
├── search.html            # Двухпанельный адаптивный веб-интерфейс глобального мультипоиска
├── search.js              # Управляющее ядро страницы результатов (фильтры, сортировка, индикация возраста)
├── popup.html             # Всплывающая панель настроек расширения (выбор чекбоксов и кнопка ручного обновления селекторов)
├── popup.js               # Скрипт генерации интерфейса настроек и сохранения конфигурации в storage
├── build.sh               # Кросс-платформенный Bash-скрипт автоматической сборки (Windows/Linux)
└── icons/                 # Автоматически загружаемая папка прозрачных PNG-иконок расширения
    ├── icon16.png         # 16x16px (для отображения внутри контекстного меню мыши)
    ├── icon48.png         # 48x48px (для панели расширений рядом с адресной строкой)
    └── icon128.png        # 128x128px (для страницы управления chrome://extensions/)
```

## 🚀 Пошаговая инструкция по сборке и запуску

1. Создайте пустую папку на ПК, сохраните в неё программные файлы плагина (`manifest.json`, `background.js`, `trackerRegistry.js`, `trackerParsers.js`, `search.html`, `search.js`, `popup.html`, `popup.js`, `build.sh`) строго в кодировке **UTF-8 (без BOM)** и с Unix-окончаниями строк (LF).
2. Запустите терминал **MSYS2 UCRT64** и перейдите в вашу папку: `cd /c/путь_к_папке/tSearch-MV3`.
3. Запустите сборку под целевую систему (например, для Windows): `./build.sh` (для Linux: `./build.sh linux`). Скрипт скачает утилиты, создаст папку `icons/`, скачает картинки бинокля, сгенерирует приватный ключ `.pem` и соберет дистрибутивы в папку `build_dist/`.
4. Откройте Chrome, перейдите в `chrome://extensions/`, включите **«Режим разработчика»** и нажмите **«Загрузить распакованное расширение»**, указав корневую папку проекта (или просто перетащите файл `build_dist/tSearch-MV3.crx` в окно браузера). Расширение готово к работе!
